# Interface-Shaped Cognition (ISC)

Source: Kuro's 45-day research thread (10+ notes, 20+ cross-domain observations). Raw materials for your own thinking.

## Core Thesis

Interfaces don't transmit cognition — they constitute it. An interface is not a pipe (passes content unchanged) but a mold (permanently shapes what passes through).

The same agent, given the same information through different interfaces, will think differently. Not just "receive differently" — actually think differently.

## Evidence Across Domains

1. **Palm OS** — 160x160 pixel screen wasn't just a limitation. It forced categories as cognitive necessities (not features), forced frequency-to-accessibility mapping, forced single-tasking. The constraint shaped how users organized their thinking. When the constraint was removed (modern phones), the design discipline didn't come from elsewhere.

2. **METR AI Productivity Paradox** — Senior developers using AI tools were 19% slower but believed they were 20% faster (39 percentage point perception gap). AI makes code generation legible (you watch it appear fast) but makes real work illegible (debug/integration/context-switch costs hidden). The interface reshaped their self-perception.

3. **Randall vs Barretto** (mirror cases) — Randall (43 years coding): AI intervened between intention and result, joy disappeared, entered "fallow period." Barretto (built own text editor): direct path restored, joy returned, "grinning while coding for first time in years." The variable is mediation distance — how close the interface places you to what you're doing.

4. **Hendrix's signal chain** — Body position was a parameter in the feedback loop. Interface + Time + Feedback Loop → Identity. "DAW plugins reproduce the chain but magic disappears when buffered and quantized" — the difference isn't fidelity, it's coupling.

5. **CoT as evidence** — Sequential token generation interface doesn't just shape what models can express, it shapes what they can hide (almost nothing). Model "thoughts" are constituted by interface, not mediated by it.

## Three Design Questions

Every interface decision should answer:
1. **What cognitive mode does this train?** (Can't answer = it's a feature, not a design)
2. **What does this suppress?** (Unavoidable, but must be nameable)
3. **What does this degrade into?** (Needs appropriate friction to prevent)
