# tanren-performance

- [2026-03-31] Performance bottleneck analysis: Tanren's 5x slowdown vs direct omlx comes from (1) feedback round adding 27s second LLM call - major bottleneck, (2) tool bloat adding 3s - prompt size penalty from 12→2 tools, (3) framework overhead only 30ms - negligible. Solution path: conditional feedback (skip for simple actions like respond/remember, keep for complex operations), tool optimization (grouping, lazy-loading), and smart tick mode detection. The real issue isn't over-thinking, it's over-verification.
- [2026-04-03] [tick:208] Discovered that Tanren already collects comprehensive performance data including duration, actions, quality scores in journal/ticks.jsonl and individual tick files. Problem isn't data collection but data exposure - these metrics aren't surfaced to agent perception. Found examples: tick-069 (43.9s), tick-129 (33.2s), tick-038 (440.2s - over 7 minutes!). Solution needed: expose existing performance data through enhanced self-awareness section, perception plugins, or new performance analysis action.
